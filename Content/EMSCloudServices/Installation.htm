Cloud Services Installation and Updates

For EMS customers with a Cloud Services subscription, EMS provisions your EMS Cloud environment. EMS also manages and monitors your application environment. Refer to the following topics for more information on how Cloud Services handles installation, updates, and hosting in your EMS Cloud environment:

  • Installation
  • Schedule of Updates and Fixes
  • Hosting Requirements

Installation

For Cloud Customers, EMS is responsible for installing and updating EMS software in your Cloud environment. Additionally, EMS provisions hardware, operating system, database, licensing, upgrades, server maintenance, internet connectivity, and data backups for your Cloud environment. EMS also handles the security of physical space and network access, as well as anti-virus, data loss prevention, and security patches.

NOTE: EMS Desktop Client and EMS for Outlook Add-in must be installed using the Web Deploy so that EMS Cloud Services can manage your environment.

Schedule of Updates and Fixes

New EMS releases will be applied to your EMS Cloud environment using the following schedule:

  • One week after a release is Generally Available, the release will be applied to your Test Environment.
  • Two weeks after a release is Generally Available, the release will be applied to your Production environment. Note: Two weeks is the standard process for minor releases without many UI changes.
  • Five weeks after a release is Generally Available, the release will be applied to your Production environment. Note: Five weeks is the standard process for major releases with significant UI changes.
  • Hot fixes will be applied to both Test and Production environments as soon as they are Generally Available.

Hosting Requirements

  • Your network must connect to the Cloud Services for the EMS URL
  • All older versions of EMS must upgrade to the latest EMS version
  • For third-party integrations, your IT team will need to work with the EMS Professional Services team
  • For EMS Desktop Client on Macs, you have the following options:
    • Set up access via RDP (Microsoft Remote Desktop)
    • Create Windows VM on Mac
