Enable User Access to EMS for Outlook

Introduction  

Concept: User Access to EMS for Outlook is set in EMS Desktop Client 

You will need to configure special settings in EMS Desktop Client in order to activate the EMS for Outlook toolbar button for Outlook users. The add-in uses each Outlook user’s EMS see Everyday User account to establish their room booking privileges based on the Process Template(s) to which the Everyday User is assigned and EMS Group(s) for which the appointment can be booked.

Tip: The optional see EMS Integration to Exchange feature enables seamless authentication for this type of user.

To enable EMS for Outlook for your Microsoft® Outlook users, you will:

  • Configure the user account (if needed) in EMS Desktop Client
  • Assign the user to a process template(s) for EMS for Outlook
  • Configure the process template to be available on EMS for Outlook

Configure EMS for Outlook Users and Process Templates 

  1. An active EMS Everyday User account must exist in EMS Desktop Client.
  2. The EMS Everyday User account must be linked to an active EMS Group record (via the External Reference field, shown below).

Tip: This topic assumes you have a working knowledge of configuration in EMS Desktop Client and highlights special tasks for enabling EMS for Outlook users. For more detailed instructions in creating new users, See Also: Configure Everyday Users.

  1. The EMS Everyday User must have at least one Delegate associated.

  1. The EMS Everyday User account must be assigned to at least one Everyday User Process Template.

  1. In the Everyday User Process Template you associate with the user, the Outlook option must be enabled.

Tip: For detailed instructions and additional Everyday User Process Template configuration options, See Also: Establish Outlook Booking Templates and Conflict Behavior.
