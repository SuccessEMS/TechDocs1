Browse Events

Limit Number of Events Displayed in Browse Events Daily View

Modified logic to ensure performance. This view is now automatically limited to 50 items when the customer has not set a value in the  "Maximum Number of Events/Day to Display in Daily View" parameter. Otherwise, observe the customer setting up to 250 items.

Fixes  

Area

Description

Authentication

New Users Not Automatically Logged in After Portal Authentication 

When parameter settings enabled users to authenticate (using a portal server) and automatically create a new account by answering User Defined Fields, they were not automatically logged in for the session and had to authenticate a second time.  

Fix: Modified how user account status is updated so that once users have entered valid information in User Defined fields, their account status becomes Active and they no longer need to authenticate for the active session.

Internal Reference # 24785
(EMS Internal Use Only)

Browse Events

Parameter Missing to Limit Number of Events in Daily View

The Daily View of the Browse Events page could display an unlimited number of events, which could cause performance issues. This was due to a parameter for V44, VEMS, which had been removed. 

Fix: Added the parameter to the EMS Web App parameters area, "Maximum Number of Events/Day to Display in Daily View," with the default set to limit the display to 50 items unless the customer has other settings in place from previous VEMS releases, in which case the former setting will be respected up to 250 items (to preserve performance). 

Internal Reference # 29895
(EMS Internal Use Only)

Request Reservation

Guest Users Able to Add Excluded Resources

When unauthenticated users requested a room, they were able to add resources that had been defined as excluded for the room.

Fix: Modified functionality to ensure that excluded resources are unavailable for selection for unauthenticated users.

Internal Reference # 29077
(EMS Internal Use Only)

Create Reservation

Keyword Search Results Incomplete In "I Know What Room I Want" Dialog

When users searched for a room by partial name, search results were incomplete.

Fix: Modified functionality to ensure that partial keyword searches show all matching results (within the parameters settings for the number of rooms to display and the user's permissions to request or reserve).

Internal Reference # 27203
(EMS Internal Use Only)

Hard Returns Fail in Special Instructions Field

When users entered Special Instructions for services under a reservation, typing a hard return in the text field did not start a new line.

Fix: Modified functionality so that hard returns result in a new line in this field as expected.

Internal Reference # 28606
(EMS Internal Use Only)

Setup Type Filter Not Enforced in List View

When guest or authenticated users searched for rooms by clicking the Let Me Search for a Room button, and a booking template or global settings were in force to filter the search by setup type, the filter was not enforced for the List view.  

Fix: Modified functionality to ensure that the setup type filter is enforced as expected in the Room search results, list view.

Internal Reference # 27406
(EMS Internal Use Only)

Edit Reservation

Viewing Booking Details Caused Internal Server Error

When users opened booking details for meetings, and multiple parameters and security settings configured booking details to show "Meeting Contact" (both Desktop Client and All Applications parameters had the Event Coordinator Title and the Group Title Singular labels set to the same Meeting Contact as the active Security template), an internal server error occurred.   

Fix: Modified functionality so that these settings no longer cause the error and booking details display as expected.

Internal Reference # 28006
(EMS Internal Use Only)

Edit Booking

Time Zone Not Adjusted Correctly For Recurrent Meeting When Changing Buildings

When users changed a building for a recurrent meeting and the building was in a different time zone, time zone adjustments incorrectly used the time zone of the original building.

Fix: Modified functionality to adjust meeting times using the timezone for the newly selected location.

Internal Reference # 28530
(EMS Internal Use Only)
