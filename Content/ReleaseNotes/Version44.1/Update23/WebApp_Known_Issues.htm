EMS Web App
Known Issue
V44.1 Update 23

Issue

EMS Internal Reference #

EMS Web App, when viewed on a mobile device, displays the EMS Mobile App download banner if either or both of the following parameters contain the default sample/starter values: 

  • System Administration > Settings > Parameters > EveryDay User Applications > Mobile Specific > URL for the EMS Mobile App in the app store
  • System Administration > Settings > Parameters > EveryDay User Applications > Mobile Specific > URL for the EMS Mobile App in the play store

The value fields for the parameters listed above may appear blank because the EMS Desktop Client ships with default sample/starter URLs in the description fields that do not display in the parameter value list until the parameters are opened and closed. Although the parameter value fields appear blank, the system recognizes the default URLs and displays the banner.

Workaround: Use the following workaround to address this issue.

  1. In the EMS Desktop Client, go to System Administration > Settings > Parameters > EveryDay User Applications tab.
  2. From the Mobile-Specific area, select the parameter URL for EMS Mobile App in the app store and click Edit.
  3. Delete the value entry and click OK.
  4. Repeat steps 1–3 for the parameter URL for EMS Mobile App in the play store.
  5. From the EMS Web App Admin Functions page, clear the cache (multiple times on a load-balanced environment).
  6. Access the EMS Web App on a mobile device and verify that the EMS Mobile App download banner no longer displays.
