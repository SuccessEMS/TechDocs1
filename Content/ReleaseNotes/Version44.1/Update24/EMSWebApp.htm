Fixes

Area

Description

Billing Reference / PO Number

 

Billing Reference and PO Number Were Not Updating

When a group was changed in the EMS Web App, the default billing reference number was not updating as expected. The old billing number remained.

Fix: Modified functionality so that the billing reference and PO numbers update appropriately.

Internal Reference # 34842
(For EMS Internal Use Only)

Maximum Number of Bookings

 

For Recurring Reservations, Maximum Number of Bookings Per Day Was Not Working

Users were allowed to book a recurring reservation when the number of rooms added were more than the maximum number allowed per the template.

Fix: Modified functionality so that users cannot add more rooms than the maximum number configured in the template.

Internal Reference # 36724
(For EMS Internal Use Only)

Online Help Link

 

Online Help Link Not Working

Users trying to access the Tech Docs help site from the EMS Web App were experiencing an error.

Fix: Modified functionality so that the help button (question mark next to the account name) in the EMS Web App takes users to the Tech Docs site.

Internal Reference # 36739
(For EMS Internal Use Only)
 

EMS Mobile-Specific URL Parameter for Displaying Mobile App Banner Not Respected

The default sample/starter url in the description field was not displaying in the parameter value list until the parameter was actually opened and closed. The Mobile-Specific URL parameter appeared blank on the parameter list, but there was actually a value entered. The EMS Web App was recognizing the default URL and displaying the banner.

Fix: Modified functionality so that the Mobile-Specific URL parameter is respected.

Internal Reference # 36860
(For EMS Internal Use Only)
 

Required Text Boxes Not Respected

In a text type user-defined field (UDF), users were able to hit the space bar in the text field and create a reservation with a required UDF containing only white space. Users should have received a warning window that a UDF is required.

Fix: Modified functionality so that users receive a warning window that a UDF is required and cannot create a reservation with white space as a UDF entry.

Internal Reference # 37046
(For EMS Internal Use Only)

Edit Reservation > Attendees

Exchange Integration Web Service

Error When Adding Attendees After Selecting Notify Changed or Notify All

For EMS Cloud Services Customers using the EMS Web App, when users edited a reservation to add attendees and then selected Notify Change or Notify All from the Notify Attendees dialog, the following error displayed: "There was a problem updating attendees on the reservation."

Fix: Modified functionality so that when users edit a reservation to add attendees to a reservation and then select Notify Change or Notify All, the error message no longer displays, and attendees are notified.

Internal Reference # 37816
(For EMS Internal Use Only)
