Fixes 

Area

Description

Login Screen

Users See an Error and Are Not Returned to the Login Screen if They Enter Incorrect Credentials

Under certain LDAP configurations, users logging into Kiosk with the incorrect credentials or a blank password saw an error and were not taken back to the login screen, leaving them unable to proceed.

Fix: Users will always return to the login screen if they enter the incorrect credentials.

Internal Reference # 32245

Session ID

 

Kiosk Session ID Vulnerability

The Kiosk session ID was not always secure, allowing external users to manipulate it in certain scenarios.

Fix: Modified functionality so that the Kiosk session ID will always be validated, preventing external users from accessing sensitive user data.

Internal Reference # 31125
